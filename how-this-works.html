How this all works.

Plain English. No jargon. Everything you need to know to work on your site.

The short version

You don't need to touch any of the tech. Just open Cowork, tell Claude what you want, and it does the work in your files. Two clicks in GitHub Desktop saves it. That's the whole thing.

What all this stuff is
The folder

Your website lives here

The folder on your computer contains two HTML files and some images. That's your website. Everything else is just tools that help you edit and back it up.

GitHub

A backup system with memory

Every time you save a change to GitHub, it remembers the full history. Nothing can ever really get lost.

GitHub Desktop is the app that syncs your folder to that backup. You only need to open it after changes are made — two button clicks and you're done.

VS Code

A text editor — you don't need it

It's like a fancier Notepad for editing HTML files. Claude handles all of that for you, so you probably won't need to open this at all.

How to work on your site
Open Cowork and start a chat with Claude

Just talk to it in plain English. No special language needed.

Tell Claude what you want

For example: "Add my story about the candidate who…" or "Change the wording in chapter 3" or "Fix the email button."

Claude edits the files directly

It writes into your folder on your computer. You can open the HTML files in your browser to see the changes.

Open GitHub Desktop

It will show you what changed automatically.

Press "Commit to main" then "Push origin"

Two clicks. That's it. Your changes are backed up and saved.
